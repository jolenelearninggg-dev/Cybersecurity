## 數位鑑識

定義：必須嚴謹且在法令規範下，用科技手法還原整個事件原貌。

目的：辨識肇事原因，且確保事後法律行動保留證據 -> 例如確保證據是未經修改的。

數位證據特性：要符合法律上的形式資格(例如不得偷錄音錄影、不得直接對證據進行操作)
用來強調證明事實，也只是輔助腳色，非絕對證據。因為也不能保證證據是「哪個人」做的，頂多證明這台電腦這個IP去入侵另外一台電腦，要詳細證明是「誰」，或許可透過監視器、指紋等其他環境證據。

### 數位鑑識種類

- storage Forensics (儲存媒體鑑識)
- Computer Forensics (電腦鑑識)
- Malware Forensics (惡意程式鑑識)
- Database Forensics => 各種帳號密碼儲存地點、交易紀錄、資料庫日誌(Transaction Logs)
- IoT => 監視器、智慧家電、穿戴式裝置紀錄
- Mobile Forensics 行動裝置 => 通話紀錄、GPS定位、通訊軟體對話紀錄
- cloude Forensics 雲端調查 => 服務上雲，資料備份上雲端 -> 怎麼樣在不影響服務的情況下，進行鑑識(API存取日至、Snapshot快照分析)
- Network Forensics => 封包分析(PCAP)、流量異常偵測
- Fraud Forensics 案件調查(詐欺、舞弊)
- Multimedia Forensics (多媒體鑑識) => 便是圖像/影片真偽，Steganography 隱寫術分析

### 作業階段
1.現場蒐證：辨識證物，防止證物受到汙染，例如被刪掉或是關機，並進行驗證(Hash)。
2.證據封存及運送：防止遭到破壞，清楚記錄證物監管鏈不中斷，且絕不對原始證物進行鑑識 -> 製作映像檔 -> 利用防寫工具(硬碟複製機)，因為只要證物被被修改，就並非原始紀錄了，無法再證明甚麼，即使找到了答案。
3.鑑識分析：分析攻擊或資安事件來源、樣貌，列出相關 **時間軸**。
  - 復原作業：還原磁碟、資料夾還原、特定檔案還原。
  - 雜湊值、特徵值、關鍵字搜尋
  - 揮發性資料、其他紙本資料
    
  > **揮發性資料及非揮發性資料**
  > >**揮發性資料**：電腦重新開機後就會消失的資料，所以要注意別拔除電源。例如：登入中帳號、已開啟的檔案、運行中程序即已啟用的服務、網路設定及連線狀態、本機分享的資源、記憶體內容及分頁檔內容 /n
  > >**非揮發性資料**：日誌檔案、裝置連接紀錄，資源回收桶、暫存檔或捷徑，網路行為歷程、系統排程作業
  > > >  `而EDR、MDR、XDR就是在記錄這些揮發性資料，在發生過程中就先記錄了`
    
4.報告及法院呈現：撰寫完整鑑識報告(證據描述、鑑識過程及發現、...)


### 分析結果要素
`只要有任何一個證據被汙染，整批證據就無法證明`
- 攻擊來源可能為個人或是小群體
- 歸類資安事件種類
- 找出攻擊者的IP及MAC，或是帳號等
- 攻擊手法及途徑，或是利用的漏洞
- 最後建立所有時間軸，並互相關聯

### 法律面

- 蒐證工具是否經過測試
- 與犯罪事實的關聯性
